# 证据与复核框架

本框架使单人维护者能够先公开透明、可复现的候选图谱，同时为未来的教育专家合作保留可审查的升级路径。它不把教材数量、模型共识或自动化检查误作教育学证明。

## 发布层级

| 发布标签 | 含义 | 最低门槛 | 不得作出的表述 |
| --- | --- | --- | --- |
| `candidate` | 基于可定位来源的候选结论 | Schema、来源、图结构和已知限制均通过检查；边有能力机制理由 | “专家验证”“权威前置标准” |
| `community-reviewed` | 经公开反馈处理的候选结论 | 满足 `candidate`，并保留意见、处理结果和冲突证据 | “等同于独立教育审核” |
| `expert-verified` | 已获独立教育审核确认的范围 | 满足 `community-reviewed`；全部 `hard` 边完成两名独立教育审核者复核 | 超出已审核范围的概括性承诺 |

`reviewStatus` 仍描述单项数据状态：`draft` 表示尚未完成维护者门禁的候选，`reviewed` 表示已完成来源、反例和自动门禁检查、可进入 `candidate` 公开版本，`approved` 表示达到专家确认的发布门槛。发布标签描述一个版本整体的证据层级，二者不能互相替代。

## 证据的作用与边界

| 证据 | 可支持的结论 | 不能单独支持的结论 |
| --- | --- | --- |
| 2022 年课程标准 | 学习目标的课程相关性、标准映射 | 教材具体安排或认知前置强度 |
| 一本教材的活动与页码 | 目标的教材出现与表征方式 | 跨版本普遍性、必要前置 |
| 多版本教材交叉检验 | 目标表述/粒度的稳健性、教材映射的差异 | 因果前置；“多数先教”不等于 `hard` |
| 教师用书、教辅、教学设计 | 课堂误解、表征、替代策略和诊断方式的线索 | 未经复核的标准映射或 `hard` 结论 |
| 教育研究 | 特定能力关系、常见困难的外部依据 | 对未覆盖人群或情境的绝对结论 |
| 独立教育审核 | 粒度、依赖强度和争议项的专业判断 | 替代来源记录或机器结构校验 |

## 单人维护的审查步骤

1. 为候选节点记录课程标准锚点、至少一个教材定位、两项可观察证据和一个诊断入口。
2. 对可获得的第二教材版本建立映射：`支持`、`表述不同`、`粒度不同`、`未出现/未定位` 四种结果均可接受，不能只记录支持项。
3. 对每条候选边写出能力机制和至少一个反例问题：学生能否借助替代策略、直观操作或不同表征达成目标？
4. 未能排除替代路径时，使用 `soft` 或暂不建边；没有独立审核时，不把边升级为 `hard`。
5. 运行自动校验，并用前置追溯、后续解锁、年级范围查询走查图是否出现荒谬关系。
6. 以 Issue 或 PR 模板公开候选、证据、反例与已知限制，记录每条有效反馈的采纳、拒绝或待定理由。

## 专家投入的最小化策略

专家时间应优先投入高风险、可复用的判断，而不是逐条抄录数据：

- 先审覆盖不同领域的 8-12 个校准节点，确定“一个可诊断学习目标”的颗粒度；
- 全量审阅拟发布的 `hard` 边，特别是跨年级、跨领域和存在替代策略的边；
- 裁决跨教材证据冲突、节点拆分/合并和标准映射歧义；
- 每次发布抽检代表性节点和边，并把结论回写为后续自动检查规则。

两名审核者应独立给出结论后再讨论分歧。审核记录保留判断、依据、日期和匿名角色标识；公开数据不保存个人身份信息。模型可生成审查包或提出反例，但不能计入任一位独立审核者。

## 接收教材与教辅的操作规则

收到材料后，维护者只记录出版信息、版本、年级册次、页码、所支持或冲突的候选 ID 及短小的独立描述。扫描件、原文转录、题目、答案和 OCR 全文留在私有输入目录，不提交仓库。每本材料先完成版权与版本登记，再进入交叉检验队列。

## 当前适用范围

北师大版二年级下册的 43 项候选仍处于 `draft`。在未完成上述交叉检验、维护者门禁和公开候选发布准备前，它们不能称为社区复核或专家确认数据。
