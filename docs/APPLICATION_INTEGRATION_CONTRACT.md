# 应用集成契约

本文件规定诊断工具、学习路径规划器、教师工具、AI 助教和示例应用如何消费本项目的公开数据。应用必须以稳定 ID、能力锚点和数据版本为接口边界，不能以教材页码、单元名、章节顺序、教材特有情境或具体表述作为业务逻辑的稳定依据。

## 稳定接口

应用可以依赖以下字段和关系：

| 接口 | 用途 | 稳定性 |
| --- | --- | --- |
| `data/manifest.json` 的 `datasetVersion` | 锁定数据兼容版本 | 发布版本内稳定 |
| `topics[].id` | 学习目标的主键、掌握状态和查询结果键 | 已发布后不复用 |
| `topics[].capabilityAnchor` | 跨版本、跨册和跨应用的能力聚合键 | 已发布后不复用 |
| `capability_anchors[].id` | 课程标准锚定能力的主键 | 已发布后不复用 |
| `topics[].standards` | 课程标准内容要求映射 | 可追加，不可静默改写既有含义 |
| `topics[].modelLayer` | 区分可诊断节点与教材实现证据 | 应用默认只消费 `diagnostic` |
| `dependencies[].topicId` / `prerequisiteId` / `strength` | 前置追溯、风险分析和学习路径 | 仅消费公开候选层的 `soft` 边 |

应用在保存用户掌握状态时，至少保存 `datasetVersion`、`topicId` 和状态时间；跨版本迁移应读取项目提供的迁移说明，而不是猜测名称相似性。

## 非接口字段

以下字段仅用于来源追溯、私有核验或展示辅助，应用不得将其作为稳定路由、分支、评分或推荐规则：

- 教材页码、册次、章节、目录和教材来源记录；
- 教材特有生活情境、例题、操作材料和算法命名；
- `sources` 中的出版社、版次或定位信息；
- 节点的中文名称和描述文字；
- `implementation_evidence` 层节点；
- 队列、社区复核和专家审核过程中的临时状态。

这些信息可能因版权边界、来源补充、节点迁移或教材版本变化而调整。应用如需展示来源，只能作为不可编辑的说明信息，不得据此推断能力关系。

## 最小查询协议

基础应用案例应支持下列查询，且输入输出均以稳定 ID 表示：

```text
prerequisites(topicId, recursive=false)
unlocks(topicId, recursive=false)
topicsByAnchor(capabilityAnchorId)
topicsByStandard(standardId)
diagnosticTopics(gradeBand)
```

返回节点时，默认过滤 `modelLayer != "diagnostic"`。应用若展示候选关系，必须同时显示数据版本和 `candidate` 证据层级，不得称为专家确认学习路径。

## 兼容性与版本

- `MAJOR`：稳定 ID、锚点语义或数据模型不兼容变更；应用必须显式迁移。
- `MINOR`：新增年级范围、节点、锚点或证据；旧 ID 继续可用。
- `PATCH`：修复来源、措辞、证据或非接口元数据；不改变稳定 ID 语义。

应用不得假设节点名称、教材定位或候选统计跨版本不变。只有 `manifest`、稳定 ID、能力锚点和迁移说明构成兼容性承诺。

## 版权与安全边界

应用不得把项目数据反向拼接为教材目录、逐页教学内容、题库或教材替代品。任何用户上传的教材、课堂材料或教辅内容应在应用层独立执行版权和隐私门禁，不能写入本仓库的公开数据层。
