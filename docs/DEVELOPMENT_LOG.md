# 项目开发日志

本日志记录会影响数据质量、项目范围、技术实现或后续决策的事实。它不是工作流水账：每一项应说明输入、结论、验证情况、风险和待办，以便新维护者能复盘“为什么这样做”。

## 记录规则

- 使用日期标题 `YYYY-MM-DD`，按时间倒序追加。
- 记录来源文件、公开链接、命令或数据版本，但不粘贴教材原文、OCR 全文或受限材料。
- 区分已验证事实、候选推断和待确认问题；模型建议只能记为候选，不是证据。
- 发生错误、返工或规则变更时必须记录原因和防范措施。
- 每个发布版本在此附上数据版本、覆盖范围、校验命令及已知限制。
