# 阶段 5.3 社区复核协议

社区复核服务于候选数据的透明改进，不等同于专家校审。所有结论维持 `candidate` 层，不能将节点标为 `approved`，也不能将依赖升级为 `hard`。

## 可接受的贡献

- 指出一个节点是否与课程标准锚点粒度不符；
- 提供不含受保护内容的最小页码或章节定位；
- 说明表述、粒度或教材实现路径的差异；
- 提供可公开的课堂反例、替代策略或标准来源；
- 对队列项目提出“支持”“待定”或“冲突待裁决”的理由。

## 禁止提交

- 教材、教辅或试题的扫描件、截图、原文、答案、OCR 或完整目录；
- 以教材先后顺序证明 `hard` 依赖；
- 将个人经验表述为专家确认；
- 包含学生个人信息的课堂材料。

## 处理流程

1. 贡献者按 `docs/review/COMMUNITY_REVIEW_TEMPLATE.md` 提交，并关联一个 `data/review_queue.json` 中的 `topicId`。
2. 维护者检查版权边界、课标锚点和复现信息。
3. 维护者将结果记录为采纳、拒绝或待定；待定项目保留原因和下一步证据需求。
4. 只有完成独立教育审核的范围才可进入 `expert-verified`；阶段 5.3 本身不触发该升级。
