# Candidate 发布检查清单

本清单适用于 `candidate-*` 发布，不要求教育专家校审。每个问题的结果应在 PR、Issue 或发布审计记录中可追溯。

## 数据门禁

- [ ] 所有公开节点与边均为 `reviewed`，没有 `draft`。
- [ ] 每个节点具备两个可观察证据、诊断提示、精确课标 ID 和至少一个可定位来源。
- [ ] 每条边均为 `soft`，有能力机制理由和至少一个可定位来源。
- [ ] `python3 scripts/validate_dataset.py` 通过。
- [ ] `python3 scripts/check_release_tree.py` 通过。
- [ ] `python3 scripts/update_manifest.py` 已执行，随后再次运行校验器。

## 证据与透明度

- [ ] 每个教材映射只保留书目信息、版本与页码，不含教材或教辅受保护原文。
- [ ] 跨教材/教辅证据登记册记录了当前可用材料、支持或冲突结论，以及尚未获得的材料。
- [ ] 发布说明写明覆盖范围、教材版本、数据状态、没有专家校审的事实和已知前置缺口。
- [ ] 所有未获独立教育审核的结论都未标为 `hard` 或 `approved`。

## 可用性验收

- [ ] 至少各执行一次前置追溯、后续解锁、年级范围查询，并保存预期结果。
- [ ] 没有无来源节点、断裂边端点、重复边、自环或环路。
- [ ] README、manifest、变更说明与数据版本一致。
