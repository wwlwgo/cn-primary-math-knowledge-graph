# 教材抽取协议

本协议用于将一册教材转化为可审核的候选学习目标，适用于北师大版及后续其他版本。教材页面是证据和定位来源，不是知识图谱的直接结构。

## 输入

每次抽取至少记录：出版社、书名、版次、出版年份、年级、册次、PDF 页码与印刷页码的对应关系。图片型 PDF 先渲染页面进行视觉核验；OCR 仅用于检索和定位，不能单独作为事实依据。

## 四层产物

| 层 | 内容 | 是否可公开 |
| --- | --- | --- |
| 原始材料 | PDF、扫描图、OCR 全文 | 否，私有核验输入 |
| 观察记录 | 单元、页码、例题/活动、可见的表征与教师提示 | 可公开简短定位，不能复制原文 |
| 候选层 | 暂定学习目标、标准映射、粒度疑问、依赖假设 | 可放入审查文档，必须标注 `draft` |
| 发布层 | 经审核的节点、边、来源和版本清单 | 是，进入 `data/` |

## 抽取步骤

1. **定位**：由目录确定单元范围，建立 PDF 页码与书页码映射。
2. **观察**：对每个新例题、活动或方法对比做视觉核验，记录它要求学生做什么，不摘录教材文字。
3. **目标拆分**：按“概念理解、程序执行、表征/建模、策略选择、语言表达”判断是否是不同能力；同一能力在不同生活情境中通常合并。
4. **标准映射**：只引用 `data/standards.json` 中已登记且有精确定位的条目。不能定位时标记为待补，不以文档级 ID 代替。
5. **依赖假设**：先写能力机制和反例，再判定 `hard` / `soft`。教材先后顺序只能作为弱证据。
6. **掌握证据**：每个保留候选给出至少两项可观察表现，其中一项应要求解释、迁移或表征转换。
7. **维护者门禁**：检查节点粒度、标准映射、来源定位、掌握证据、边理由和反例问题；完成 Schema 与图校验后，项目可从候选层进入明确标注为 `candidate` 的公开数据。
8. **独立复核（升级路径）**：第二位教育审核者检查粒度、标准映射、证据和依赖强度；所有拟升级为 `hard` 的边必须双审。此步骤是进入 `expert-verified` 的条件，不阻塞 `candidate` 发布。
9. **发布转换**：候选版只发布完成维护者门禁的数据；专家确认版只发布同时满足独立教育审核要求的数据。两者均须通过自动校验并附带相应证据层级说明。

## 粒度判据

应拆分：学生可能“会算但不会建模”“会操作但不能解释”“会一种表征但不能转换”的能力。

应合并：同一运算或概念仅改变故事情境、人物、物品或数值大小的重复练习。一个例题中的多个动作也不应自动拆成多个节点，除非每个动作可独立诊断或会解锁不同后续学习。

## 失败模式

- 用 OCR 的错字或漏字推断知识点。
- 把章节标题、习题编号或综合复习页当作微知识点。
- 把例题展示的两种方法自动写成两条 `hard` 依赖。
- 用“教材前面出现过”代替依赖理由。
- 将课程标准的大主题直接当作节点，而不拆成可观察目标。
