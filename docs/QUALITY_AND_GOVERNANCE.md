# 质量与治理

## 自动门禁

每次数据修改必须运行：

```bash
python3 scripts/validate_dataset.py
```

校验器检查 JSON、必填字段、ID 唯一性、边端点、重复边、自环、有向环和 `manifest` 计数。发布流程还应检查 Schema 合规、可复现的校验日志和变更说明。

## 证据层级与人工门禁

仓库可以发布 `candidate` 数据，但发布说明必须明确其为“有来源、机器验证、待教育复核”的候选结论。候选发布至少要求来源可定位、Schema 和图校验通过、每条关系有能力机制理由，并公开已知限制或争议；它不以专家校审为前置条件。

`expert-verified` 数据须经教育审核。教育审核至少检查：目标是否有恰当颗粒度；掌握证据是否可观察；标准映射是否准确；边是否有能力机制依据；`hard` 是否被过度使用。每批至少抽样审阅 10% 的节点和边，所有 `hard` 边全量双审。

人工资源优先顺序为：全部拟发布的 `hard` 边、跨年级或跨领域边、节点拆分/合并、证据冲突项，以及每批覆盖不同领域的校准样本。模型不能作为独立教育审核者。

## 跨版本交叉检验

北师大版、人教版等教材，以及依法可引用的教师用书、教学设计或教辅材料，可以作为同一学习目标的独立观察来源。它们用于发现表述、粒度或教学路径的差异，并增强节点存在或教材映射的证据；它们不能单独证明认知前置关系，更不能以多数教材的编排顺序替代依赖理由。

当不同材料一致时，记录为交叉支持；当存在差异时，维持候选状态并审查是否需要拆分、合并或改写节点。不得只挑选支持既有结论的材料，冲突来源也须在维护记录中说明。

## 变更政策

新增、拆分、合并、废弃节点都需保留变更说明。公开版本中的 ID 不得重用。发现会误导学习诊断的错误时，应尽快修复并发布补丁版本；若该错误影响关系含义，需提供旧 ID/边的迁移记录。
