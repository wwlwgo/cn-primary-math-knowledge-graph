# 数据标准

权威机器契约在 `schemas/`。本文件解释其教育含义和编写规则。

## 节点

`id` 使用稳定、可读的格式 `cn_math_g<首次引入年级>_<英文短名>`，例如 `cn_math_g2_three_digit_addition_no_carry`。年级表示首次系统引入的典型年级，不表示学生只能在该年级学习。

`type` 可取 `CONCEPTUAL`、`PROCEDURAL`、`REPRESENTATIONAL`、`LANGUAGE`、`STRATEGIC`、`META`。一个节点只选最主要的认知性质；若同一名称同时包含概念和程序，应拆成两个目标。

`evidence` 必须是可观察的学生表现，避免“理解”“熟练”等不可判定词。至少一条应检查迁移或解释，而不只检查计算答案。`assessmentPrompt` 是诊断入口，不等同于正式测验题。

`standards` 是课程标准映射，不是教材页码。`sources` 至少包含一条可复查的定位记录；教材来源记录页码、版本和书目信息，标准来源记录章节/条目。

`capabilityAnchor` 连接节点与课程标准导出的稳定能力锚点。`modelLayer` 为 `diagnostic` 时，节点可参与候选图谱与跨版本节点比较；为 `implementation_evidence` 时，节点仅保留为教材情境、操作或算法偏好的历史证据，不参与跨版本节点重合率，也不得作为依赖边端点。

## 依赖边

边对象中 `topicId` 是目标，`prerequisiteId` 是前置。禁止自环和重复边。图必须是有向无环图。

判定 `hard` 前，审核者要能回答：缺少此前置时，学生是否无法理解、表征或执行该目标的核心要求？若能使用合理替代策略或教学支架达成，则通常应为 `soft`。`reason` 需说明能力机制，例如“目标要求按位相加，因而须能辨认十进制位值”，不能只写“教材先学”。

## 审核状态

- `draft`：抽取候选，尚未完成维护者的来源、反例和自动门禁检查，不能进入公开版本。
- `reviewed`：已完成维护者的结构化复核、来源定位和自动门禁，可进入明确标注为 `candidate` 的公开版本；仍可修改，且不表示独立教育审核已经完成。
- `approved`：满足 `expert-verified` 发布门槛；`hard` 边须两名独立教育审核者同意。

审核身份和讨论记录放在维护流程中，不在公开节点中保存个人信息。
